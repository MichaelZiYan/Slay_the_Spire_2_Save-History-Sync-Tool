# 杀戮尖塔 2 (Android) 存档提取成功路径指南

本指南用于从**未 Root 的小米手机**中提取《杀戮尖塔 2》存档，以便与 Steam 版进行同步。

## 🛠️ 前置准备

* **环境**：Windows 10/11 PC，已安装 **Java (JRE/JDK)**。
* **工具包**：
* `adb.exe` 及相关 DLL。
* `abp.jar` (Android Backup Extractor)。


* **工作区**：在磁盘根目录创建一个**无空格、无特殊字符**的文件夹（例如 `G:\sts_sync`），并将上述工具放进去。

---

## 🚀 核心步骤

### 1. 手机端备份

在小米手机上：**设置 -> 更多设置 -> 备份与恢复 -> 手机备份 -> 本地备份**。
只勾选《杀戮尖塔 2》（Slay the Spire 2），开始备份。

### 2. 拉取备份文件到电脑

连接手机并开启 USB 调试，在工作区文件夹打开 **CMD**，运行：

```cmd
adb pull "/sdcard/MIUI/backup/AllBackup/[最新日期文件夹]/Slay the Spire 2(com.megacrit.slaythespire2).bak" test.bak

```

> **注意**：文件名中包含括号和空格，务必用双引号包裹路径。

### 3. 剥离小米私有文件头

小米的 `.bak` 文件头部包含私有数据，直接解压会报错。使用以下 **PowerShell 流式处理命令** 将其转换为标准 `.ab` 格式（此方法不占用额外内存）：

```powershell
powershell -Command "$in = [System.IO.File]::OpenRead('test.bak'); $out = [System.IO.File]::Create('test.ab'); $buf = New-Object byte[] 1024; $in.Read($buf, 0, 1024) | Out-Null; $magic = [System.Text.Encoding]::ASCII.GetBytes('ANDROID BACKUP'); $off = -1; for($i=0; $i -le 1010; $i++){ $match=$true; for($j=0; $j -lt $magic.Length; $j++){ if($buf[$i+$j] -ne $magic[$j]){ $match=$false; break } }; if($match){ $off=$i; break } }; if($off -ne -1){ $in.Seek($off, [System.IO.SeekOrigin]::Begin) | Out-Null; $in.CopyTo($out); Write-Host '成功：起点位于 '$off } else { Write-Error '未找到备份头' }; $in.Close(); $out.Close()"

```

### 4. 转换为可读取的 Tar 包

使用 Java 调用 abp 工具：

```cmd
java -jar abp.jar unpack test.ab test.tar

```

> 如果设置了备份密码，请在提示时输入。

### 5. 提取存档文件

使用 **Bandizip** 或 **7-Zip** 打开 `test.tar`，定位到：
`apps\com.megacrit.slaythespire2\f\default\1\profile1\saves\`

---

## 📂 存档文件说明

| 文件名 | 功能描述 | 同步建议 |
| --- | --- | --- |
| **`progress.save`** | **全局进度**。包含解锁项、进阶等级、成就。 | **必须同步** |
| **`current_run.save`** | **当前对局**。记录你玩到一半的那把游戏。 | **建议同步** |
| **`prefs.save`** | **系统设置**。音量、分辨率等。 | 视设备情况选择 |
| **`history/`** | **历史战绩**。过去所有对局的详细记录。 | 建议同步文件夹 |

---

## ⚠️ 常见坑点总结

1. **路径避坑**：不要在包含 `[]`、`()` 或中文字符的文件夹路径下运行脚本，否则 Java 和 PowerShell 极易报错。
2. **内存溢出**：小米备份通常很大（本例约 977MB），避免使用简单的数组读取，应使用 `FileStream` 流式拷贝。
3. **权限拒绝**：在 Android 11+ 上，不要尝试用 `adb shell cd /data/data`，那是死胡同，必须通过“系统备份”绕路。
